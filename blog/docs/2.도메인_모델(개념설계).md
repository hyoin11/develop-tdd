# 도메인 개념 설계

### 1. 개요<br>
   블로그 서비스의 주요 도메인(Entity)와 관계를 정리<br>
   요구사항 정의서를 기반으로, 각 엔티티의 속성과 관계를 개념적으로 설계

### 2. 엔티티 정의
1. 회원 (users)
   - 설명 : 서비스에 가입한 사용자 관리
   - 속성
     - 회원 순번 (pk)
     - 이메일 (아이디, unique)
     - 비밀번호
     - 닉네임
     - 상태 (active / inactive)
     - 가입일시
     - 수정일시
     - 탈퇴일시

2. 게시글 (posts)
   - 설명 : 회원이 작성하는 블로그 게시글
   - 속성
     - 게시글 순번 (pk)
     - 회원 순번 (fk)
     - 제목
     - 글내용
     - 생성일시
     - 수정일시
     - 삭제일시

3. 댓글 (comments)
   - 설명 : 게시글에 작성되는 댓글. 대댓글 구조를 지원할 수 있음
   - 속성
     - 댓글 순번 (pk)
     - 게시글 순번 (fk)
     - 회원 순번 (fk)
     - 내용
     - 부모 댓글 순번(대댓글을 위함) (nullable, fk)
     - 생성일시
     - 수정일시
     - 삭제일시

4. 좋아요 (likes)
   - 설명 : 회원이 게시글에 남기는 좋아요
   - 속성
     - 좋아요 순번
     - 게시글 순번
     - 회원순번
     - 생성일시

### 3. 관계 정의
- 회원:게시글 (1:N)
- 회원:댓글 (1:N)
- 회원:좋아요 (1:N)
- 게시글:댓글 (1:N)
- 게시글:좋아요 (1:N)
- 회원:게시글 (좋아요를 통해 N:M)

### 4. 참고
- '좋아요' 기본은 row 삭제 방식 사용
- 향후 사용자 행동 이력 추적 및 통계 확장을 위해 상태값 관리로 전환 가능
