# 도메인 개념 설계
- 회원
  - 회원 순번
  - 이메일 (아이디)
  - 비밀번호
  - 닉네임
  - 상태
  - 가입일시
  - 수정일시
  - 탈퇴일시
- 게시글
  - 게시글 순번
  - 회원 순번
  - 제목
  - 글내용
  - 생성일시
  - 수정일시
  - 삭제일시
- 댓글
  - 댓글 순번
  - 게시글 순번
  - 회원 순번
  - 내용
  - 부모 댓글 순번(대댓글을 위함)
  - 생성일시
  - 수정일시
  - 삭제일시
- 좋아요
  - 좋아요 순번
  - 게시글 순번
  - 회원순번
  - 생성일시
<br><br>
- 관계
  - 회원:게시글 (1:N)
  - 회원:댓글 (1:N)
  - 회원:좋아요 (1:N)
  - 게시글:댓글 (1:N)
  - 게시글:좋아요 (1:N)
  - 회원:게시글 (좋아요를 통해 N:M)

#### 참고
- '좋아요' 기본은 row 삭제 방식 사용
  - 향후 사용자 행동 이력 추적 및 통계 확장을 위해 상태값 관리로 전환 가능
