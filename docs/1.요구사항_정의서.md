# 블로그 서비스

### 1. 개요
- 프로젝트 목적<br>
  블로그 컨텐츠 플랫폼을 개발하여 CRUD, 검색, 캐싱, 배치, 모니터링(APM) 등 백엔드 기술을 학습
- 개발 범위<br>
  MVP 기능에서 출발하여 점진적 확장을 고려
  (예. 댓글 대댓글, 검색 자동완성, 추천 알고리즘 등)

### 2. 사용자 시나리오
- 사용자는 회원가입을 통해 계정을 생성할 수 있다.
- 로그인 후 JWT 토큰을 받아 게시글을 작성할 수 있다.
- 게시글은 다른 사용자가 읽고, 댓글과 좋아요를 남길 수 있다.
- 사용자는 검색 기능을 통해 원하는 게시글을 찾을 수 있다.
- 인기글은 하루 단위 조회수를 기준으로 확인할 수 있다.
- APM을 통해 서비스 성능과 에러율을 모니터링 할 수 있다.

### 3. 기능 요구사항
   1. 회원 / 인증
      - 회원가입 (이메일, 비밀번호, 닉네임)
      - 로그인 (JWT 발급 및 검증)
      - 추후 확장: 소셜 로그인 (OAuth2, 비밀번호 재발급)

   2. 게시글
      - 게시글 생성, 조회, 수정, 삭제
      - 추후 확장: 게시글 상태 관리 (공개 / 비공개 / 임시저장), 태그, 카테고리 기능
   
   3. 댓글
      - 댓글 작성, 수정, 삭제
      - 추후 확장: 대댓글, 댓글 좋아요

   4. 좋아요
      - 게시글 좋아요 및 취소

   5. 검색
      - 제목 / 본문 검색 (Opensearch)
      - 추후 확장: 자동완성, 태그 검색, 인기 검색어 제공

   6. 인기글
      - 하루 단위 조회수 집계 (Redis 캐싱 -> 배치 작업으로 DB 반영)
      - 추후 확장: 주간 / 월간 인기글, 사용자 맞춤 인기글?

   7. 모니터링 (APM)
      - HTTP 요청 / 응답 시간, DB 쿼리, Redis / 외부 API 호출 성능 추적
      - 오류 발생 시 에러율 및 Stack Trace 기록
      - 서비스 전체 트래픽, 응답시간, 오류율 대시보드 제공
      - 추후 확장: 알람 설정 (슬랙 / 이메일 연동)

### 4. 비기능 요구사항
- 보안: JWT 인증, 비밀번호 암호화
- 성능: Redis 캐싱 활용 (조회수, 인기글, 세션 관리)
- 확장성: Opensearch 검색, Spring Batch 처리
- 테스트: TDD: JUnit5, Testcontainers 기반 통합 테스트
- 배포: Docker 기반 빌드 & AWS 배포
- 모니터링: Elastic APM Agent + APM Server + Opensearch

### 5. 기술 설계
   1. 빌드 & 배포 파이프라인
      1. Spring Boot 빌드
      2. Dockerfile 작성 및 이미지 빌드
      3. Docker 이미지 AWS ECR에 Push
      4. AWS EC2 환경에 Pull & Deploy
      5. GitHub Actions를 통한 자동 빌드 & 배포
   
   2. 인프라 구성 요소 역할
      - EC2: 애플리케이션 서버 (Spring Boot 실행)
      - RDS (MySQL): 영구 데이터 저장 (회원, 게시글, 댓글 등)
      - Redis: 캐싱 서버 (조회수, 인기글, 세션)
      - OpenSearch: 검색 엔진 (검색, 자동완성, 집계)
      - Elastic APM: 애플리케이션 메트릭 / 트레이스 수집
      - S3 (추후): 이미지 / 파일 업로드 저장소

### 6. 향후 확장 계획
- 소셜 로그인 (google, github 등)

- 관리자 페이지 (게시글 / 댓글 관리 기능)
- APM 알람 (Slack, 이메일)
